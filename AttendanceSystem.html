Project Overview

A web-based Smart Attendance Management System that automatically validates employee check-ins and check-outs using GPS coordinates matched against predefined office geofence zones. The system supports multiple branches, role-based access, shift scheduling, email notifications, and HR reporting — all in a single unified application.

Core Concept: Employees click a button on their device — the app captures their GPS location and sends it to the server. Laravel calculates the Haversine distance between the employee's position and the branch geofence. If inside the zone, the attendance is logged. If outside, it is rejected with an error. All geofence logic runs server-side so it cannot be spoofed from the client.

Geofence Attendance — How It Works

All geofence validation happens in Laravel — never on the client. This prevents coordinate spoofing from the browser. The GPS only provides raw coordinates — a human-readable address is fetched separately via reverse geocoding.

Full Flow — Step by Step

1. Button ClickedEmployee clicks "Clock In" on their device (phone or laptop browser)
2. GPS PermissionBrowser prompts for location access — employee must allow it once
3. Coordinates FetchedBrowser Geolocation API returns { latitude, longitude } using GPS chip (high accuracy mode)
4. Reverse GeocodingReact calls OpenStreetMap Nominatim API with the coordinates — gets back a human-readable address like "15 Main Blvd, Lahore, Pakistan" — stored alongside the log
5. Sent to LaravelReact sends { gps_lat, gps_lng, readable_address } to POST /api/attendance/checkin with auth token
6. Branch FetchedLaravel reads the authenticated employee's assigned branch — gets its center lat/lng and radius_meters
7. Haversine RunsLaravel calculates the straight-line distance in meters between the employee's GPS and the branch center
8. Decision MadeDistance ≤ radius → attendance logged. Distance > radius → request rejected with distance shown in the error message
9. Flag AppliedIf valid, check-in time is compared to shift start time — flagged as on-time or late automatically

The Browser Function — navigator.geolocation.getCurrentPosition()

This is a built-in function available in every modern browser — no installation, no library, no API key needed. You simply call it and the browser handles everything internally. It decides whether to use the GPS chip, nearby Wi-Fi networks, or cell tower signals to determine the device's location, picks the most accurate source available, and returns the coordinates back to your code.

Source Used by BrowserAccuracyHow It Works
GPS Chip~5 metersCommunicates with satellites in orbit to pinpoint location
Wi-Fi~20–50 metersMatches nearby Wi-Fi network names against a global database of known router locations
Cell Towers~300–1000 metersTriangulates position based on which mobile towers the device can detect

You never write code to handle any of these sources. The browser chooses automatically. The only thing the user sees is a one-time popup: "Allow this site to access your location?" — once allowed, the coordinates are returned instantly.

The function takes three arguments: a success callback (runs when location is found), an error callback (runs if the user denies or GPS fails), and an options object where enableHighAccuracy: true tells the browser to prefer the GPS chip over Wi-Fi for better precision.

Step 3 — GPS Capture in React

const handleClockIn = () => {
  navigator.geolocation.getCurrentPosition(
    (position) => {
      const { latitude, longitude } = position.coords;
      getAddressThenSubmit(latitude, longitude);
    },
    (error) => { alert("Location access denied. Please enable GPS."); },
    { enableHighAccuracy: true, timeout: 10000 }
  );
};

enableHighAccuracy: true forces the device to use the GPS chip instead of Wi-Fi triangulation, giving a more precise location reading.

Step 4 — Reverse Geocoding (How Address Is Fetched)

GPS coordinates are numbers — { lat: 31.5204, lng: 74.3587 } — they do not include a street name or city. To display and store a readable location, React calls the free OpenStreetMap Nominatim API. No API key is required.

const getAddressThenSubmit = async (lat, lng) => {
  const res = await fetch(
    `https://nominatim.openstreetmap.org/reverse?lat=${lat}&lon=${lng}&format=json`
  );
  const data = await res.json();
  const address = data.display_name;
  // e.g. "15 Main Boulevard, Gulberg III, Lahore, Punjab, Pakistan"

  sendClockIn(lat, lng, address); // now send all three to Laravel
};

Step 7 — Haversine Formula in Laravel

The Haversine formula calculates the shortest distance between two points on the Earth's surface given their latitude and longitude. It accounts for Earth's curvature. Result is in meters.

private function haversine($lat1, $lng1, $lat2, $lng2): float
{
    $earthRadius = 6371000; // meters

    $dLat = deg2rad($lat2 - $lat1);
    $dLng = deg2rad($lng2 - $lng1);

    $a = sin($dLat / 2) ** 2
       + cos(deg2rad($lat1)) * cos(deg2rad($lat2))
       * sin($dLng / 2) ** 2;

    return $earthRadius * 2 * atan2(sqrt($a), sqrt(1 - $a));
}

Step 8 — Check-In Controller Logic

public function checkIn(Request $request)
{
    $employee = $request->user();
    $branch   = $employee->branch;

    $distance = $this->haversine(
        $request->gps_lat, $request->gps_lng,
        $branch->lat, $branch->lng
    );

    if ($distance > $branch->radius_meters) {
        return response()->json([
            'success' => false,
            'message' => "You are {$distance}m away. Must be within {$branch->radius_meters}m."
        ], 403);
    }

    AttendanceLog::create([
        'employee_id'      => $employee->id,
        'branch_id'        => $branch->id,
        'type'             => 'check_in',
        'gps_lat'          => $request->gps_lat,
        'gps_lng'          => $request->gps_lng,
        'readable_address' => $request->readable_address,
        'timestamp'        => now(),
        'is_valid'         => true,
        'flag'             => now()->gt($branch->shift_start) ? 'late' : 'on_time'
    ]);
}
