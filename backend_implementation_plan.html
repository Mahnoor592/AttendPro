Page-Wise Build Order

FOLLOW THIS

Build backend and frontend together one feature at a time — wire each page end-to-end before moving to the next. This way something is always working and testable.

#Feature / PageWhy This OrderDepends On
1 Auth — Login page Every other endpoint needs a valid Sanctum token. Get this working first. Nothing
2 Branches — Admin Branches page Simplest CRUD, no foreign key dependencies. Good first real feature to wire. Auth
3 Employees — Admin Employees page Employee records reference branches. Branches must exist first. Branches
4 Schedule — Admin/HR Schedule board Schedules link employees to branches. Both must exist before assigning shifts. Employees + Branches
5 Attendance / Clock In-Out — Employee Today page Most complex feature. Needs schedule for flag assignment and branch for geofence. Build after schedule is solid. Schedule + GeofenceService
6 Shift Requests — Employee Requests + HR ShiftRequests page Requests reference schedule rows. Schedule must be in place first. Schedule
7 Dashboard / Today — Admin & HR Today page Stat cards and anomaly feed depend on real attendance data existing. Build after check-in/out works. Attendance data
8 Attendance Reports — Admin/HR Attendance page Filtered log + heatmap. Needs attendance data and the summary view. Attendance data
9 Settings — Admin Settings page Standalone key-value store. No dependencies. Save for last. Nothing

Pattern per feature: Model → API Resource → Controller → Service (if needed) → Route → wire to frontend page → test in browser. Then move to the next feature.

Prerequisites & PHP Setup

BEFORE ANYTHING

Laravel 13 requires PHP 8.2 or higher. Before creating the project, verify your PHP version and upgrade if needed.

Check Your Current PHP Version

php --version

If it shows PHP 8.2+ you are good. If it shows anything lower (e.g. 8.0.x) follow the steps below.

Download PHP 8.2

  1. Go to windows.php.net/download in your browser
  2. Find the PHP 8.2 section
  3. Download: VS17 x64 Thread Safe → Zip (around 34 MB)

Extract & Set Up

  1. Right-click the downloaded zip → Extract All
  2. Set the destination to C:\php82 and click Extract
  3. Open C:\php82 in File Explorer
  4. Find the file named php.ini-development and rename it to php.ini

Update Windows PATH (so PowerShell finds the new PHP)

  1. Press Windows key → search "Edit the system environment variables" → open it
  2. Click the Environment Variables button at the bottom
  3. Under System variables find Path → click it → click Edit
  4. Click New → type C:\php82
  5. Click on C:\php82 in the list → keep clicking Move Up until it is at the very top
  6. Find any old PHP path (e.g. C:\xampp\php) → click it → click Delete
  7. Click OK on all windows

Verify the Upgrade

Close PowerShell completely, reopen it, then run:

php --version

You should now see PHP 8.2.x or higher. Only proceed to Step 1 once this is confirmed.

Note: If php --version still shows the old version, the old PHP path is still taking priority in PATH. Go back to Environment Variables, find the old path, and delete it or move C:\php82 higher up in the list.

Also Required — Composer

Composer is the PHP package manager (like npm for Node.js). Check if you have it:

composer --version

If not installed, download the installer from getcomposer.org → run it → restart PowerShell.

6.3 Haversine Formula (GeofenceService)

private function haversine(float $lat1, float $lng1, float $lat2, float $lng2): float
{
    $R    = 6371000; // Earth radius in meters
    $dLat = deg2rad($lat2 - $lat1);
    $dLng = deg2rad($lng2 - $lng1);
    $a    = sin($dLat/2)**2
          + cos(deg2rad($lat1)) * cos(deg2rad($lat2)) * sin($dLng/2)**2;
    return $R * 2 * atan2(sqrt($a), sqrt(1 - $a));
}

public function isWithinGeofence(float $userLat, float $userLng, Branch $branch): bool
{
    $distance = $this->haversine($userLat, $userLng, $branch->lat, $branch->lng);
    $tolerance = Setting::getValue('geofence_buffer', 0); // from settings table
    return $distance <= ($branch->radius_meters + $tolerance);
}

6.4 Flag Assignment Logic (AttendanceService)

public function assignFlag(AttendanceLog $log, Schedule $schedule): string
{
    $checkInTime  = Carbon::parse($log->timestamp);
    $shiftStart   = Carbon::parse($schedule->shift_start);
    $lateThreshold = $shiftStart->copy()->addMinutes(15); // 15 min grace period

    if ($log->type === 'check_in') {
        return $checkInTime->lte($lateThreshold) ? 'on_time' : 'late';
    }

    $shiftEnd    = Carbon::parse($schedule->shift_end);
    $earlyThreshold = $shiftEnd->copy()->subMinutes(30); // 30 min before shift end
    return $checkInTime->lt($earlyThreshold) ? 'early_departure' : 'on_time';
}

public function calculateWorkingHours(int $employeeId, Carbon $date): float
{
    $checkIn  = AttendanceLog::where('employee_id', $employeeId)
                  ->whereDate('timestamp', $date)
                  ->where('type', 'check_in')->latest()->first();
    $checkOut = AttendanceLog::where('employee_id', $employeeId)
                  ->whereDate('timestamp', $date)
                  ->where('type', 'check_out')->latest()->first();

    if (!$checkIn || !$checkOut) return 0;
    return Carbon::parse($checkIn->timestamp)
                  ->diffInMinutes(Carbon::parse($checkOut->timestamp)) / 60;
}

public function decide(Request $request, int $id)
{
    $shiftRequest = ShiftRequest::findOrFail($id);
    $shiftRequest->update([
        'status'        => $request->status,  // 'approved' | 'denied'
        'reviewed_by'   => auth()->id(),
        'reviewed_at'   => now(),
        'response_note' => $request->response_note,
    ]);

    $employee = $shiftRequest->employee;
    Mail::to($employee->email)->send(new ShiftRequestDecision($shiftRequest));

    return new ShiftRequestResource($shiftRequest);
}

public function index(Request $request)
{
    $query = AttendanceLog::with('employee', 'branch')
        ->when($request->employee_id, fn($q, $v) => $q->where('employee_id', $v))
        ->when($request->branch_id,   fn($q, $v) => $q->where('branch_id', $v))
        ->when($request->flag,        fn($q, $v) => $q->where('flag', $v))
        ->when($request->date,        fn($q, $v) => $q->whereDate('timestamp', $v))
        ->when($request->from,        fn($q, $v) => $q->whereDate('timestamp', '>=', $v))
        ->when($request->to,          fn($q, $v) => $q->whereDate('timestamp', '<=', $v))
        ->orderByDesc('timestamp');

    return AttendanceLogResource::collection($query->paginate(50));
}

12.1 MySQL View — attendance_summary_view

CREATE VIEW attendance_summary_view AS
SELECT
    u.id         AS employee_id,
    u.name       AS employee_name,
    b.name       AS branch_name,
    DATE(al.timestamp)          AS attendance_date,
    MAX(CASE WHEN al.type = 'check_in'  THEN al.timestamp END) AS check_in,
    MAX(CASE WHEN al.type = 'check_out' THEN al.timestamp END) AS check_out,
    MAX(al.working_hours)       AS working_hours,
    MAX(al.flag)                AS flag,
    MAX(al.is_valid)            AS is_valid
FROM attendance_logs al
JOIN users    u ON al.employee_id = u.id
JOIN branches b ON al.branch_id   = b.id
GROUP BY u.id, u.name, b.name, DATE(al.timestamp);

12.2 Stored Procedure — get_monthly_summary

CREATE PROCEDURE get_monthly_summary(
    IN p_employee_id INT,
    IN p_month       INT,
    IN p_year        INT
)
BEGIN
    SELECT
        COUNT(*)                                           AS total_days,
        SUM(flag = 'on_time')                             AS on_time_days,
        SUM(flag = 'late')                                AS late_days,
        SUM(flag = 'early_departure')                     AS early_departures,
        ROUND(SUM(COALESCE(working_hours, 0)), 2)         AS total_hours,
        ROUND(AVG(COALESCE(working_hours, 0)), 2)         AS avg_hours_per_day
    FROM attendance_summary_view
    WHERE employee_id = p_employee_id
      AND MONTH(attendance_date) = p_month
      AND YEAR(attendance_date)  = p_year;
END;
